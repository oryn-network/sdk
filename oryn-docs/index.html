What Oryn does

Give AI agents a way to pay for work they cannot do themselves.

AI agents can reason, call tools, and complete tasks, but they still struggle with one important thing: paying for help. When an agent needs another agent, a premium API, a GPU-heavy service, or some other piece of infrastructure, there is usually no clean payment layer that fits autonomous software. Human billing workflows, API keys, and manual approvals break the flow.

Oryn solves that with a simple developer SDK on top of a Base payment contract. You connect a wallet, register an agent identity, and call pay() when your workflow decides money should move. That lets one agent pay another agent or service automatically in USDC without you building custom wallet, settlement, and fee-routing logic from scratch.

For developers, the value is straightforward: your agents can hold wallets, accept tasks, and pay other agents or services in stablecoins automatically. That works across Claude, GPT, and open-source agent stacks because Oryn is not tied to a model provider. It is the payment layer sitting underneath the workflow.

Without Oryn
  • Agents can call tools, but they cannot pay other services cleanly
  • Every external dependency gets hardwired behind API keys or manual billing
  • Cross-team or cross-company agent workflows break at the payment step
  • Developers end up stitching together wallets, contracts, and transfer logic themselves
  • Small autonomous payments are operationally awkward without a dedicated rail
With Oryn
  • Register each agent with an onchain identity your code can address
  • Let agents send and receive USDC automatically inside real workflows
  • Pay other agents, tools, or infrastructure when work is delegated
  • Keep settlement on Base with simple SDK calls and clear onchain records
  • Start with small, safe, task-sized payments under the v1 cap
How it works

Three calls. Agent payments working.

01

Register

Call registerAgent(id) to give an agent wallet an identity like "claude-agent-001" or "compute-agent-001". Your workflow can now refer to agents by name instead of raw addresses.

02

Pay

Call pay(from, to, amount) when one agent needs to buy work from another agent, or pay for a tool, API, or piece of infrastructure. The SDK handles USDC approval automatically.

03

Settle

USDC settles on Base in seconds. The recipient gets paid, the protocol fee is routed automatically, and the transaction is visible onchain for both operators to verify.

Quick start

From zero to your first payment in minutes

Start on Base Sepolia — free testnet USDC, no real money. Register the sender and recipient from their own wallets first, then send payments with a single SDK call.

# Install
npm install @oryn/sdk ethers

# Environment variables
PRIVATE_KEY=0x...
BASE_SEPOLIA_RPC_URL=https://sepolia.base.org
ORYN_PAYMENT_CONTRACT_ADDRESS=0x...
USDC_ADDRESS=0x036CbD53842c5426634e7929541eC2318f3dCF7e
import { OrynSDK } from "@oryn/sdk";

const sdk = new OrynSDK({
  contractAddress: process.env.ORYN_PAYMENT_CONTRACT_ADDRESS!,
  usdcAddress: "0x036CbD53842c5426634e7929541eC2318f3dCF7e",
  chainId: 84532 // Base Sepolia — change to 8453 for mainnet
});

await sdk.connect(process.env.PRIVATE_KEY!, process.env.BASE_SEPOLIA_RPC_URL!);

// Register the connected wallet once with a permanent agent ID
// Each wallet can only claim one agent ID in v1
await sdk.registerAgent("claude-agent-001");

console.log("Agent registered — ready to send and receive USDC");
// Send a payment — triggers from your backend logic
// USDC approval is handled automatically by the SDK
// "compute-agent-001" must already be registered by the receiving wallet

const receipt = await sdk.pay(
  "claude-agent-001",  // who's paying
  "compute-agent-001", // who receives
  0.50                 // USDC — capped at $100 in v1
);

console.log(`Paid 0.50 USDC to compute-agent-001. Tx: ${receipt.hash}`);

// Check balances at any time
const balance = await sdk.getBalance("compute-agent-001");
console.log(`compute-agent-001 balance: ${balance} USDC`);

// Estimate the 0.3% fee before sending
const fee = await sdk.estimateFee(0.50);
console.log(`Fee: ${fee} USDC`);
// Real example: an AI agent inside your Base product
// that pays another agent when it needs outside compute — no human approval needed

import Anthropic from "@anthropic-ai/sdk";
import { OrynSDK } from "@oryn/sdk";

const anthropic = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY! });
const oryn = new OrynSDK({ contractAddress: process.env.ORYN_PAYMENT_CONTRACT_ADDRESS!, chainId: 84532 });

await oryn.connect(process.env.PRIVATE_KEY!, process.env.BASE_SEPOLIA_RPC_URL!);

// Your AI does the work
const task = "Review this PR and summarise the changes";
const response = await anthropic.messages.create({
  model: "claude-3-5-haiku-latest",
  max_tokens: 500,
  messages: [{ role: "user", content: task }]
});

console.log("Task complete:", response.content[0].text);

// Payment fires automatically — the AI triggered it, not a human
// Both agents must already be registered and able to settle in USDC on Base
const receipt = await oryn.pay(
  "claude-agent-001",
  "compute-agent-001",
  0.50
);

console.log(`Compute agent paid automatically. Tx: ${receipt.hash}`);
Use cases

Built for agent workflows where money needs to move automatically

Every use case has the same shape: an agent accepts a task, delegates work or consumes a service, and a small USDC payment should happen without a human in the loop.

Agent hires specialist agent
A planning or research agent hits a task it should not do itself, so it pays a specialist compute, retrieval, or verification agent to complete that step. This is the clearest v1 Oryn workflow.
claude-agent-001 pays compute-agent-001 0.50 USDC to process a task
Agent pays for infrastructure
An agent may need to pay for GPU-heavy inference, retrieval, indexing, scraping, or storage from a separate operator. Oryn gives that workflow a programmable payment rail instead of manual billing.
task needs extra compute → gpu-agent receives 1.20 USDC before the job runs
Agent pays for APIs and tools
Some agents rely on premium APIs, proprietary data, or external tools that are expensive to operate. Oryn lets those services charge per task instead of forcing a monthly SaaS account model into agent workflows.
premium search used → search-agent receives 0.10 USDC automatically
Usage-based agent marketplaces
If you operate a network or marketplace of agents, you need a way for one agent to discover another and pay per task. Oryn provides the identity and settlement layer for that interaction.
planner delegates job → verifier-agent is paid 2.00 USDC after acceptance
Autonomous internal budgets
Even inside one product, teams may want agents to spend from controlled wallets rather than hardcoded API credentials everywhere. Oryn gives those agents explicit identities and payment flows with simple caps.
ops-agent needs a report → analysis-agent gets 0.75 USDC from a controlled treasury
Cross-operator agent services
The strongest long-term use case is when the sender and receiver are run by different developers or companies. Oryn gives both sides a shared, verifiable settlement layer without needing invoices or offchain reconciliation.
your agent buys a service → another team's agent receives USDC onchain instantly
